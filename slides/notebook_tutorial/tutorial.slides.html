What you need¶

Anaconda python distribution installed. Get it here: https://www.anaconda.com/download/

You can also make your own virtual environments. Conda is easier if you:

  • don't have administrative rights on your machine (but see containers tomorrow)
  • are new to the command line and like to have a graphical interface (make sure you get Anaconda, not miniconda)
  • don't want to or can't deal with installing system dependencies (conda gets most for you

What is a notebook¶

A Jupyter notebook is an interactive way to run code through your browser

Easy to annotate code and findings with markdown (great for sharing). Also great for reproducing your work. See also https://nbviewer.jupyter.org/

Why do I want a notebook¶

You can run code on a remote machine.

  • no need to move data around
  • only need a web browser
  • good for that last minute analysis while traveling to a conference

How do I install¶

Don't use your system python. Use Conda (virtual) environments.

  1. Open a new terminal
  2. Create a new conda environment for jupyter notebook: conda create --name jpserver
  3. Check the available conda environments: conda env list. You should see the new jpserver environment in the list.
  4. Activate the new environment source activate jpserver
  5. This is an empty conda environment, with nothing in it: conda list --name jpserver. We need to install python first: conda install --name jpserver python

How do I install jupyter notebook¶

Use Conda environments.

  1. Now we can install the jupyter packages in this environment: conda install --name jpserver jupyter jupyterlab
  2. Move to your home directory or some other directory that you would like the server to have access to: cd ~/
  3. Start the jupyter lab server jupyter lab

This should open a new browser window with the jupyter lab layout.

Get a conda environment for your project¶

Don't run everything in your jupyter environment.

  1. Create a new conda environment for your current project (e.g. brainhack) but this time request that python is installed directly: conda create --name brainhack python. You can confirm that python was in fact installed (with some additional tools) by running conda list --name brainhack
  2. Now install ipykernel so we can make jupyter aware of your new environment: conda install --name brainhack ipykernel
  3. Activate the environment with source activate brainhack
  4. Register your new environment with jupyter: python -m ipykernel install --user --name brainhack --display-name "python brainhack"

Go back to your notebook server and check for the new kernel

You can install non-python kernels as well¶

The nice thing about jupyter is that it can run a long list of interpreters. Have a look here for a long list.

Install packages in your project environment¶

  1. Ask conda to install the packages (e.g. numpy and matplotlib): conda install --name brainhack numpy matplotlib.
  2. Go to your jupyter lab website that is running in the jpserver environment and start a new notebook with the brainhack kernel.
  3. Confirm that you can access the newly installed packages by running import numpy as np and executing the cell (Shift + Enter). If you don't get an error message you are good to go

Install packages not available through conda¶

You have two options here:

  1. See if the package is available from a conda channel. These channels are additional repositories maintained by community members. I don't know how reliable they are.
  2. Install them directly from the Python Package Index. The Python Package Index or pypi is a central hosting platform for python packages. Most packages are available from there. You can install them using pip..
    • When you use pip, you need to first activate your environment: source activate brainhack
    • pip install nilearn nibabel

Pitfalls¶

There are two main things to look out for:

  • jupyter cells share variable names. That means the output of your cells can change depending on the order in which you run them
In [1]:
message = 'Jupyter is cool'
print(message)

Jupyter is cool
In [3]:
message = 'Brainhack is cool'
In [6]:
print(message)

Brainhack is cool
  • You should never try to install python packages from inside the notebook. See this post for a long explanation.

Summary¶

Jupyter notebooks are great for:

  • trying out things
  • seeing results immediately
  • (poor man's) debugging
  • annotating results
  • sharing results
  • running things on a remote machine

Not great for:

  • version control
  • strictly linear execution
